# trading-dashboard

Website dashboard repository for the trading system.

Repository relationship:
- trading repos keep their own output/report generation responsibilities
- `trading-dashboard` is responsible for the downstream website/dashboard surface only

## What this repository does

`trading-dashboard` is the dashboard/UI repo for the trading stack.
It is responsible for:
- website dashboard construction
- dashboard UI/page/module design
- local server / adapter layer for dashboard use
- dashboard-facing data contracts
- shared dictionary/display metadata used by the dashboard
- visualization of outputs already generated by the trading repos

## What this repository does not do

`trading-dashboard` should not own:
- upstream market-data acquisition
- strategy replay computation
- model-training or ranking internals
- live execution internals
- cross-repo scheduling/timing policy
- cross-repo canonical report generation

In other words:
- trading repos generate their own outputs/reports
- `trading-dashboard` visualizes those outputs in the website dashboard

## Core architecture rule

Do not migrate report-generation responsibility out of the producer repos into this repo.
This repo should stay focused on dashboard construction and visualization.

## Practical baseline

Use the old `ops-dashboard` implementation/content as the practical baseline for this repo direction.
